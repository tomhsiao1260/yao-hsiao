# Cursor 使用心得

這是我第一次跟 Cursor 這樣的 AI 程式編輯器互動，對我來說是個蠻奇妙的體驗。

Agent 的內建功能很快地吸引了我，他讓我能透過自然語言進行大面積的更改，程式碼就像流體那樣動態的流動著。當我想編輯一些程式碼細節，他往往做出預測，讓我滿意地按下 tab 鍵。此外，文檔的自動撰寫也補足了我過去的短板。在數小時的使用後，我發現了兩大與這類系統的互動方式：
```
1. 想像一個新功能，然後叫 Cursor 在現有的程式碼加入它

2. 想像手邊程式碼底下某個應用的子情境，然後叫 Cursor 刪減程式碼到剛好能滿足那樣的情境

這兩者剛好相反，一個是建構，一個是拆解，前者是想打造時會做的事，後者是想理解時會做的事
```

這其實正是[我過去撰寫方式](https://yao-hsiao.vercel.app/explore)的延續，不過輪廓變得更加清晰了。對我來說，理解和打造其實是一體兩面的，就像物理學家費曼說的：
```
What I cannot create, I do not understand.
```

隨著這類工具的進步，除錯的時間變短了，能思考的時間也變多了。以前想好一個功能，接著就是漫長的除錯，但現在想清楚要打造的功能後，問題就解掉大半了；以前想理解一個系統，會先想好一個應用的子情境，接著就是漫長的拆解和除錯，但現在只要想清楚，Agent 就能幫你拆解到那樣的情境，進一步加速了理解的過程。

對了，在用 Cursor 時還想到了一個迷因 :P
```
過去: ctrl+c, ctrl+v

現在: tab, tab, tab, ...
```
