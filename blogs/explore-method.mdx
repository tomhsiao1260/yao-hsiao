# 探索新事物的方法

這篇文是為了整理過去幾年來觀察到的一種方法論：
```
透過拆解來理解一個系統，並基於這份理解，打造出新的東西。
```

下面會舉許多例子，以不同的視角來觀察這套方法論，以及它是怎麼在不同情境和尺度下作用。我會把重點放在這些例子之間的相似性，而不是歸納出精簡的結論。另外，文章會有點長，你能針對自己感興趣的段落閱讀，因為這些例子都是彼此獨立的。

## 程式的撰寫

我在學習程式的時候，常需要看別人的程式碼，尤其是電腦圖學的領域。對我來說，那是一個又一個的黑盒子，等待我去解密。出於好奇，我會開始拆解這些程式，理解一些中間狀態的演進，直到將專案刪減到能被充分理解的最簡形式，我便能以此為基礎，打造出自己想要的其他應用。如果有興趣，[這篇文章](https://yao-hsiao.vercel.app/explore)有更多實踐細節的描述。

直到像 Cursor 這樣的 AI 編輯器出現，我跟計算機的互動方式發生了轉變，但用的還是同一套方法。Cursor 能拆解程式碼到我想理解的情境，也能在某個基礎之上打造新的功能。如果有興趣，可以看[這篇文章](https://yao-hsiao.vercel.app/cursor)。

## 知識的梳理

在知識的學習上，Heptabase 創辦人詹雨安也用到了類似的方法。他認為如果要深度理解一個主題，就要將外部知識拆解成一個個原子化的概念卡片，並基於這些概念，重新建構出彼此的關聯，並認為這是個理解、內化知識的過程。另外，[這篇文章](https://wiki.heptabase.com/the-best-way-to-acquire-knowledge-from-readings)有更多的實踐細節。

## 第一性原理

Elon Musk 認為第一性原理是套很有用的物理工具，並把它用在創業、工程和決策上。他認為第一性原理就是把事物拆解到最基本、最有可能是真實的原子層級。然後從這些基礎開始，盡可能有說服力地向上推理，並避免透過隱喻或分析的方式向上推理建構。

對他來說，這個拆解的過程，是種接近真實的方法。

## 相機的構造

數位相機透過光學鏡頭、光圈和快門，將周遭的環境資訊層層篩選，只有一小部分落在了感光元件上，接著轉為電訊號，並透過處理器重建圖像，最終顯示在相機的螢幕上。

感光元件成為了現實和數位的橋樑，一邊是將環境資訊進行拆解，另一邊是根據這些真實資訊，重新建構出一張圖像。

## 侏羅紀公園

沒有人看過恐龍，那侏羅紀公園裡栩栩如生的恐龍，又是基於什麼建構出來的？

牠們是根據過去所挖掘的化石推敲出來的，而這些化石是經過地球數億年的篩選所存留下來的。
