# Vesuvius Challenge 一週年的感想

Vesuvius Challenge 這個挑戰滿一年了，我也在這個過程中拿到了不少開源獎項，所以想藉這個機會寫下我當下的感受和學到了什麼。

首先，非常高興看到 Vesuvius Challenge 官方將我的名字列入一份分工圖表裡。剛好也是個機會讓我可以回顧我對這個挑戰的看法。對我來說，這項科學挑戰代表了許多人不斷嘗試和探索的成果。有些方法能取得了成功，有些則失敗了，但有時候一個方法的出現也可能啟發其他人在另一個方向上努力。這樣的體驗經常在社群中互相討論中出現。從我的角度來看，我第一次聽說這個挑戰是在去年三月，那時候整個挑戰社群還很小，挑戰者之間互相認識。

印象最深刻的是 Luke 在早期創建了一個 Volume 視覺化的雛形。我從中得到了一些啟發，於是就開始開發 Segment Viewer。雖然後來他改轉向機器學習的方向去解這個挑戰，但如果沒有他早期的嘗試，我可能不會因此得到啟發來開發相關工具。

再來就是 Casey 的突如其來的發現促使了 Luke 開發相關的機器學習辨識工具，也主導了未來幾個月大家的研究方向。我印象最深刻的一點是 Casey 憑藉自己的強大視覺力在龐大的資料中找到了 crackles 的線索。當時應該沒什麼人會這麼做，主流的方法是嘗試使用在 Kaggle 上表現良好的算法，但這條路一直難有所突破。Casey 選擇花費大量時間仔細檢視資料，嘗試找出那些若有似無卻具有潛力的線索，他的毅力也讓我印象深刻。

還記得當時 Casey 公佈他的發現時，整個社群的討論尚不明朗，Luke 是第一個非常重視 Casey 結果的人，並很快開始嘗試建立相關的算法，這段也使我對 Luke 在研究上的直覺印象深刻。當時我就在思考，這個發展肯定超出了官方的預期，因為這意味著我們可能需要放棄原本 Kaggle 上的成果，另外開發一套模型。結果，Youssef 在之後幾個月裡花了大量時間比較不同模型的優缺點，成功將大家在 Kaggle 上的努力與 Casey 他們的貢獻相結合。

還記得在那段時間裡，我進行了許多與視覺化相關的嘗試。這些工具也讓 Segment 團隊更了解哪些區域尚未被切割，而他們的切割進展也有助於機器學習辨識能力的提升。然後，Djosey 嘗試將這些文字拼貼在一起也給了我不少啟發，促使我與另一個高中生 Fish 一起開發了 VC Whiteboard，這個合作過程也讓我學到了很多。

上面提到的這些過程也讓我體會到，這個挑戰的成功以及最終 Grand Prize 的誕生都是一系列插曲和社群間不斷嘗試的結果。如果中間有任何一個環節的缺失，或者社群中缺少任何一個看似不起眼的嘗試或失敗，後續的發展可能都會完全不同。

<br />

另一個讓我印象深刻的是，官方或者說前 Github 創辦人 Nat Friedman，他在發起這個挑戰上的時程規劃得蠻縝密的。回顧了他的計劃後，我重新思考了去年參加挑戰的一些過程，有種彷彿以另一個視角重新審視了同一件事情的感覺。

例如，為了加速社群中的研究進展，官方會定期提供階段性的開源獎項，這種做法不僅定期更新了參賽者之間的研究進展，還避免了挑戰者為了爭取更長遠的 Grand Prize 而推遲發表成果。而且 Nat 他們每週都會思考一個問題：為了最大程度地提高某個人贏得 Grand Prizes 的機會，現在可以做的最佳事情是什麼？同時，他們還給了 Segment 團隊一個重要目標，那就是要與參賽者密切交流或提供幫助。

聽到這些點時，我當下想到的是如果能更早了解這些思路，就能更清楚知道社群裡其他區塊的人想要什麼，我在什麼位置，以及他們的目標等等，這樣我就更有動力專注在真正重要的方向上。好比說，如果我能早點知道 Nat 和 JP 他們每週都在思考如何提高某人贏得年底 Grand Prizes 的機會，那我可能就會更專注在幫助整個社群獲得 Grand Prize 上，執行上也比較不容易失焦。又或是如果我早點知道 Segment 團隊的目標是與我們密切合作，我可能就會更勇於向他們提問，而不是擔心他們會感到困擾而錯過寶貴的機會，這也會促使我更主動地將 Segment 團隊的資源用於工具的開發。

此外，我也對官方拆解問題的方式印象深刻，他們不僅提供了具體但不過於制式的需求清單，讓我可以在自己感興趣的方向上自由探索。長遠上，Nat 也清楚知道每個階段應該做些什麼。舉例來說，直接挖掘整棟高級別墅進行重建工作或尋寶等點子聽起來很有趣，但成本和不確定性都過高，也很難得到外部資金的注入。相反，如果能先找出別墅裡的卷軸，修復尚未出土的古代文獻，首先只修復一卷，然後再把這些經驗套用到另外三卷，接著再把剩下的一千卷都挖掘出來，最後再來把整棟別墅重建出來。這樣循序漸進的過程反而能持續吸引各種領域有趣的人才，資金也隨著社群的進展逐漸注入，Nat 對解決這一系列複雜問題的切入點我覺得蠻有趣的。

而且到目前爲止，單一卷軸的辨識仍然是一項耗時的工作，而 Nat 正好利用這段時間規劃醞釀之後更龐大的挖掘和掃描工作，進而逐步累積到足夠的資金來數位重建整個豪華別墅。這種逐步拆解一個願景並實踐的方法讓我印象深刻。另外我覺得每個階段的社群的里程碑都蠻重要的，因為這些都是下一階段進展的基礎，透過這個過程挑戰者間也可以持續互相學習。

<br />

最後，讓我最驚喜的就是 Vesuvius Challenge 官方的 JP 在最近來到台灣跟我們見面這件事，我真的沒想到他會來這裡。
仔細想想，如果我去年沒有持續參加這個挑戰，JP 就不會來台灣，也就不會見到 Alan，我也不會知道 Paper Program 的故事，這塊島嶼應該就不會跟 Vesuvius Challenge 有所交集。感覺我也在這個世界的洪流中改變了點什麼，也認識了來自世界各地對這個挑戰感興趣的人。所以要用一句話描述我對這個挑戰的想法，我會說如果是獨自一人，一個很難的問題可能會讓人望之卻步，但這些問題卻能很自然的將一群喜歡挑戰的人聚集在一起，彼此學習並試圖解決這些難題。這就是我參加 Vesuvius Challenge 一年後的感受！
